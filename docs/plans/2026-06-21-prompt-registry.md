# 프롬프트 레지스트리 + 슈퍼어드민 콘솔 구현 계획 (SPEC-AI-008)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 마케팅 블로그 프롬프트(시스템·GEO규칙·출력스펙)·모델·temperature를 코드 상수에서 DB로 옮기고, 슈퍼어드민 콘솔에서 CRUD·버전·활성화·플레이그라운드로 재배포 없이 튜닝한다.

**Architecture:** ai-server에 `prompt_override` seam을 깔고(없으면 `geo_rules.py` 폴백) → 게이트웨이가 `ai_prompt` DB에서 active 프롬프트를 로드해 생성 요청에 주입 + 어드민 CRUD/활성화/preview 제공 → web `(console)` 콘솔이 편집·활성화·플레이그라운드 UI. 정적 부분(프롬프트·모델)만 DB, 동적 데이터(키워드·말투샘플·매장맥락)는 기존처럼 게이트웨이 코드 주입.

**Tech Stack:** ai-server(FastAPI/Python 3.12/uv/pytest/ruff), api(Spring Boot/Kotlin/JPA/ktlint), web(Next.js/TS/React/eslint).

## Global Constraints

- ai-server: Python 3.12, `uv run ruff check .`(E/F/I/UP/B/ASYNC, line-length 120) + `uv run pytest` 통과. async I/O. 사용자 동적 입력은 `[USER INPUT — DATA ONLY]` 펜스 유지.
- ai-server는 **stateless** — DB 직접 접근 금지. `prompt_override`는 게이트웨이(X-Internal-Key 신뢰)만 전달.
- api: Kotlin, **FK 미사용**(간접참조), **수동 DDL**(`docs/sql/migration/`, Flyway 없음, ddl-auto=validate), **soft delete**(`deleted_at`), `id BIGINT GENERATED BY DEFAULT AS IDENTITY`, `update_updated_at()` 트리거. `START TRANSACTION;...COMMIT;` + rollback 파일. `./gradlew ktlintCheck test` 통과.
- api 어드민 엔드포인트는 전부 `@RequiresAdmin` 게이팅.
- web: Next.js `(console)` 슈퍼어드민 그룹, `npm run lint && npx tsc --noEmit && npm run test` 통과.
- 모델 화이트리스트: `claude-haiku-4-5`, `claude-sonnet-4-6`(LiteLLM 등록 모델). 그 외 거부.
- **폴백 불변식**: `ai_prompt`가 비어도/게이트웨이 로드 실패해도 블로그 생성은 `geo_rules.py` 기본값으로 동작.
- 문서·커밋 한국어, conventional commits, 변경 파일만 `git add`, `Co-Authored-By: Claude <noreply@anthropic.com>`.
- 채널당 active 1개 불변식(`uq_ai_prompt_active_per_channel`).

```python
    def build_messages(self, gen_input: BlogGenInput) -> list:
        from app.agents.marketing.geo_rules import default_blog_prompt

        base = default_blog_prompt()
        ov = gen_input.prompt_override
        system_md = (ov.system_md if ov and ov.system_md else base["system_md"])
        rules_md = (ov.rules_md if ov and ov.rules_md else base["rules_md"])
        output_spec_md = (ov.output_spec_md if ov and ov.output_spec_md else base["output_spec_md"])

        instruction_parts = ["다음 조건으로 네이버 블로그 초안을 작성하세요."]
        # ... (말투 샘플 블록 — 기존 유지) ...
        instruction_parts.append(rules_md)   # GEO_RULES 상수 → rules_md
        # ... (키워드/상황/메모/매장맥락 — 기존 유지) ...
        instruction_parts.append(output_spec_md)  # OUTPUT_SPEC 상수 → output_spec_md
        content: list = [{"type": "text", "text": "\n\n".join(instruction_parts)}]
        # ... (사진 — 기존 유지) ...
        return [SystemMessage(content=system_md), HumanMessage(content=content)]  # BLOG_SYSTEM 상수 → system_md
```
